Entra endpoints

  • GET /.well-known/openid-configuration
  • GET /{tenant}/.well-known/openid-configuration
  • GET /discovery/keys / /discovery/v2.0/keys
  • POST /token and Entra-compatible token paths
  • GET /authorize (interactive flow)
  • POST /login

Test endpoints

  • Endpoint to retrieve a test token with defaulted (if not configured) claims plus any query parameters as claims
    GET /test-tokens/{tenantId}/{appId}/{clientId}?scope=read,scope=write
  • Endpoint for just signing provided token
    POST /test-token/sign?token=token

These are described here but are not implemented yet.
